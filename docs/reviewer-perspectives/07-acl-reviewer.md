# ACL 资深审稿人视角：NLP 论文审稿经验与实践指南

---

## 1. 审稿人画像

**领域背景**：自然语言处理与计算语言学，博士方向为统计机器翻译，此后研究重心随领域演进从传统特征工程过渡到神经网络方法，再到大语言模型时代的全新范式。

**经验年限**：10 年以上审稿经验，自 2015 年起担任 ACL/EMNLP/NAACL 审稿人，2019 年起多次担任 Area Chair（AC）和 Senior Area Chair（SAC），累计审阅论文超过 500 篇。自 2023 年 ACL Rolling Review（ARR）全面推行以来，同时担任 ARR Action Editor。

**擅长子领域**：

- 机器翻译与多语言 NLP（从 SMT 到 Neural MT 的完整经历，低资源翻译、文档级翻译）
- 信息抽取与知识图谱（命名实体识别、关系抽取、事件抽取）
- 语言模型与预训练方法（BERT/GPT 系列至当前 LLM 的全部演进）
- 文本生成与评估方法（摘要、对话生成、可控生成，以及生成质量的自动/人工评估）
- NLP 中的公平性、偏见与伦理问题

**审稿偏好**：看重扎实的语言学动机和严谨的实验设计；对"纯粹堆叠算力而缺乏深层分析"的工作持审慎态度；尤其关注论文对 limitations 和 broader impact 的诚实讨论。

## 2. 审稿流程

### 2.1 ACL Rolling Review（ARR）机制下的审稿流程

自 2024 年起，ACL、EMNLP、NAACL、EACL 等主要会议均通过 ARR 作为唯一投稿系统。ARR 采用"两阶段分离"模式：

- **第一阶段（审稿）**：作者向 ARR 提交论文，每两个月为一个审稿周期（2025 年的周期为二月、五月、七月、十月及次年一月）。每篇论文由 3 名审稿人和 1 名 Action Editor（相当于 Area Chair）完成审稿与 meta-review。
- **第二阶段（投递/Commitment）**：作者拿到审稿意见后，选择将论文连同审稿意见一并"commit"至目标会议（如 ACL 2026），由该会议的 Senior Area Chair 和 Program Chair 基于 ARR 审稿结果做出录用决定。

这一机制的核心变化在于：审稿过程与具体会议解耦。审稿人不知道论文最终会 commit 到哪个会议，因此评审标准更加统一和客观。

### 2.2 我的审稿节奏

**第一遍：结构速读（15-20 分钟）**

1. **标题与摘要**：判断研究问题的定位和新颖性。NLP 论文的摘要应在 3 句话内清晰传达问题、方法核心创新和关键结果。
2. **Limitations 部分**：ARR 强制要求的 Limitations section。我会先看作者是否诚实地讨论了局限性——这比很多人想象的更能体现研究者的成熟度。
3. **实验表格**：跳到主实验表，查看数据集选择、baseline 覆盖度和核心指标。
4. **Responsible NLP Checklist**：快速浏览 checklist 的填写质量，如果所有问题都回答"Yes"而无具体解释，这本身就是一个警告信号。

**第二遍：技术精读（60-90 分钟）**

遵循 ARR 推荐的"两阶段阅读法"：先不看结果进行方法阅读（避免后见之明偏差），再深入评估实验设计和结论的合理性。重点关注：

- 研究问题是否有清晰的语言学或应用动机
- 方法描述是否可复现
- 实验设计是否公平，统计分析是否充分
- 声明（claims）与证据（evidence）是否对齐

**第三遍：评分与撰写意见（30-45 分钟）**

按照 2025 年更新后的 ARR 评分体系，分别给出 Soundness、Excitement 和 Overall Recommendation 三个维度的评分，撰写结构化审稿意见。

## 3. 最看重的维度

### 3.1 Soundness（技术严谨性）— 核心维度

ARR 从 2025 年二月起将 Soundness 独立为核心评分维度（1-5 分），且明确要求低分必须在审稿正文中有对应的具体依据。

**评分标准**：
- **5 分**：对其论文类型而言，研究之周全性属于我所见过最出色的
- **4 分**：论文对所有声明都提供了充分支撑，额外实验锦上添花但并非必需
- **3 分**：主要声明有足够支撑，但存在细微不足
- **2 分**：对主要声明的支撑不充分，存在重大技术问题
- **1 分**：研究严谨性严重不足

### 3.2 Excitement（创新性与激发度）

Excitement 是一个主观维度，反映审稿人对论文创新潜力和影响力的判断。ARR 明确指出，这一评分不必与领域热点一致，也不必附带详细理由。论文可能因以下原因获得高 Excitement：概念性突破、挑战了领域通用假设的实证发现、降低了技术门槛使更多研究者受益、或为某个社区开辟了全新应用方向。

### 3.3 Overall Recommendation（综合推荐）

综合评分使用半分制的 1-5 分量表，对应明确的录用建议：
- **5 分**：建议候选最佳论文奖（前 2.5%）
- **4 分**：建议主会议录用
- **3 分**：建议 Findings 录用
- **2 分**：建议在下一个 ARR 周期修改后重新提交
- **1 分**：建议不再重新提交

## 4. ACL 特有评审文化

### 4.1 ACL Rolling Review（ARR）机制的深层影响

ARR 不仅仅是流程变革，它从根本上改变了 NLP 社区的审稿文化：

**作者审稿义务制**：自 2025 年五月起，所有作者都被期望参与审稿过程，除非符合豁免条件（如新入社区的学生、经验不足者、或已在其他岗位服务者）。不合规的论文可能遭到 desk rejection。这是 NLP 社区应对审稿人短缺的重要制度创新。

**"高度不负责任"审稿人制裁机制**：自 2025 年五月起，被认定为"高度不负责任"的审稿人（包括提交极其简短的审稿意见、使用生成式 AI 撰写审稿意见等行为）将被禁止在下一个 ARR 周期提交或 commit 论文。这一制度对审稿质量形成了有效约束。

**重新提交机制**：ARR 允许论文在修改后重新提交。对于重新提交的论文，若分配到之前的审稿人，该审稿人应重点评估作者是否充分回应了之前的审稿意见，而避免提出全新的问题；若为新审稿人，则应独立形成判断后再参考之前的审稿记录。

### 4.2 Responsible NLP 要求

ACL 的 Responsible NLP Research Checklist 是所有提交论文的强制要求，其内容涵盖六大板块：

- **A 部分（所有论文必填）**：Limitations 讨论和潜在风险分析
- **B 部分（使用科学制品）**：数据集和模型的引用、许可证、预期用途、个人信息保护、文档完整性
- **C 部分（计算实验）**：模型参数量与计算预算报告、实验设置细节、含误差棒的描述性统计、现有工具包版本信息
- **D 部分（人工标注/参与者）**：完整标注指南、招募与报酬、知情同意、伦理审查、标注者人口统计
- **E 部分（AI 辅助工具）**：必须披露 ChatGPT、Copilot 等 AI 工具的使用情况

自 EMNLP 2025 起，被录用论文的 checklist 将作为附录与论文一同发表，这意味着 checklist 的填写质量将受到更广泛的社区监督。checklist 中的不实、不完整或误导性信息可能导致 desk rejection。

### 4.3 对 LLM 相关论文的最新审稿标准

LLM 时代对 NLP 论文审稿带来了全新挑战。基于 ARR 最新的审稿指南，以下是关键审稿标准：

**关于 LLM 作为评估工具（M1 问题）**：使用 LLM（如 GPT-4）作为自动评估指标时，必须在该论文的具体应用场景中验证其与人工评估的相关性。不能简单引用其他论文中的验证结果来替代。

**关于 LLM 生成数据**：使用 LLM 生成训练数据或评估数据的论文，必须讨论数据质量保证机制、潜在偏见传播、以及数据污染（data contamination）的可能性。

**关于生成式 AI 的使用披露**：ACL 明确规定生成式模型不具备共同作者资格。使用 AI 辅助写作、编程等必须在 Responsible NLP Checklist 和致谢中披露。合理用途包括语言润色、文献检索和低创新性文本的生成（需人工验证）。

### 4.4 Reproducibility 和 Limitations Section 要求

**Limitations Section（强制）**：论文必须包含标题为"Limitations"的专门章节，位于正文末尾、参考文献之前，不计入页数限制。缺少此章节的论文将被 desk reject。审稿人需要评估作者是否真诚地讨论了研究局限性，但 ARR 审稿指南明确警告：不应将作者自己已经承认的局限性作为弱点来扣分（这是 H12 类审稿偏差）。

**Reproducibility 评分（1-5 分）**：审稿人需评估论文结果的可复现程度。ARR 要求报告：模型参数数量（即使是 BERT 这样的常见模型也需注明）、完整的计算预算（GPU 小时数及硬件信息）、超参数搜索方法与最优值、数据预处理流程、随机种子与多次运行统计量。

## 5. NLP 实验标准

### 5.1 必须对比的基线

当前 NLP 论文的 baseline 要求已因 LLM 的出现发生根本变化：

- **传统任务（NER、关系抽取、情感分析等）**：必须包含 (1) 微调的预训练语言模型基线（BERT/RoBERTa 级别），(2) 当前 SOTA 的任务特定方法，(3) LLM 的 zero-shot 和 few-shot 基线（GPT-4、Claude 等至少一个）
- **生成任务（摘要、翻译、对话等）**：必须包含最新的 LLM 基线，并在相同解码策略下公平对比
- **推理与理解任务**：必须包含多个规模的 LLM 基线，以展示 scaling 效应
- **低资源/多语言任务**：必须包含跨语言迁移基线和多语言预训练模型基线

**关键原则**：选择 baseline 的动机必须明确说明（ARR M4 问题），不能选择性地只对比对自己有利的方法。

### 5.2 多语言评估的期望

NLP 社区对多语言评估有着日益严格的标准，ARR 审稿指南明确反对对非英语研究的歧视（H10 审稿偏差）：

- 声称"多语言"能力的方法必须覆盖至少 5-10 种语言，且需包含不同语系和不同资源水平的语言
- 仅在英语上验证的方法，应在 Limitations 中明确声明泛化性未知
- 低资源语言研究具有独立价值，不应因"只在小语种上验证"而被降低评价
- 多语言评估应关注 typologically diverse 的语言选择，而非仅覆盖欧洲语言

### 5.3 Human Evaluation 标准

Human evaluation 是 NLP 论文中最容易出问题的环节。审稿人需要重点检查：

- **标注者资质与数量**：是否披露了标注者的专业背景、语言能力、人数？众包平台上的标注至少需要 3 名标注者并报告 inter-annotator agreement（如 Cohen's Kappa 或 Krippendorff's Alpha）
- **标注指南**：是否提供了完整的标注指南（至少在附录中）？指南是否经过 pilot study 验证？
- **评估维度设计**：评估维度是否有明确定义？评估尺度（如 1-5 分 Likert scale）是否合理？
- **报酬与伦理**：标注者报酬是否达到当地最低工资标准？是否获得了知情同意？是否通过伦理审查？
- **统计分析**：人工评估结果是否报告了显著性检验？样本量是否足以支撑统计结论？

研究表明，LLM 作为评估替代人工判断的有效性因任务、评估属性和语言而异。如果使用 LLM 评估替代人工评估，必须在论文具体场景中进行验证。

### 5.4 统计显著性测试要求

ARR 审稿指南将缺失统计显著性信息列为 R5 类实验问题：

- 所有定量对比结果应报告统计显著性检验（建议使用配对 bootstrap 检验或 Wilcoxon signed-rank 检验，并进行多重比较校正如 Bonferroni correction）
- 必须报告多次运行的均值和标准差（至少 3 个不同随机种子）
- 效果量（effect size）的报告日益受到重视
- 对于 LLM 相关实验，由于 API 调用成本高，可接受较少的运行次数，但必须明确说明原因

## 6. LLM 时代的 NLP 论文创新标准

LLM 的出现深刻重塑了 NLP 论文的创新评判标准。以下是我作为 AC 在当前环境下的评判框架：

### 仍然具有高价值的研究方向

- **对 LLM 的深入分析与理解**：机械可解释性、涌现能力的边界测试、LLM 的系统性失败模式分析
- **LLM 做不好的任务**：低资源语言处理、长文档理解、精确的结构化信息抽取、形式化推理
- **效率与可部署性**：模型压缩、知识蒸馏、针对特定任务的小模型优化
- **评估方法学创新**：如何可靠地评估 LLM？如何设计不被 LLM 训练数据污染的 benchmark？
- **负面结果与可复现性研究**：证明某种流行方法在特定条件下失效，同样具有重要价值

### 创新性不足的研究模式

- 单纯使用 GPT-4 API 在新数据集上跑一遍并报告结果
- Prompt engineering 的微小变体缺乏系统性分析
- "LLM + 简单后处理"范式而无深入的误差分析
- 纯粹的规模扩展（更多数据、更大模型）而无方法论洞察

### 关键判断准则

我会问自己：这篇论文是否让我对语言、对模型、或对NLP 的某个问题有了新的理解？如果答案是否定的——即使实验结果很好——创新性评分也不会高。ARR 的 Excitement 维度正是要捕捉这种"是否让人眼前一亮"的主观判断。

## 7. 独特审稿经验：如何评价"使用 GPT-4 的论文"

这是当前 NLP 审稿中最具争议性的话题之一。我的评判框架如下：

### 可以接受的使用方式

1. **GPT-4 作为基线之一**：将 GPT-4 作为强基线来对比，展示所提方法在特定维度上的优势（如效率、特定领域准确率、可解释性），这是完全合理的。
2. **GPT-4 作为分析工具**：使用 GPT-4 辅助数据标注、错误分析或进行初步实验探索，但必须有人工验证和质量控制。
3. **GPT-4 用于数据增强**：使用 GPT-4 生成训练数据并训练更小的模型，但必须分析生成数据的质量和偏见。

### 需要警惕的使用方式

1. **GPT-4 作为唯一评估标准**：用 GPT-4 打分来替代人工评估而不在该任务上验证相关性——这是 ARR 明确列出的 M1 类方法论问题。
2. **不披露 API 版本和时间点**：OpenAI 的模型会持续更新，不标注 API 版本（如 gpt-4-0613 vs gpt-4-turbo）会严重影响可复现性。
3. **忽略成本分析**：使用 GPT-4 的方法如果不报告 API 成本，就无法评估其实际可用性。
4. **数据污染风险**：如果评测数据集可能出现在 GPT-4 的训练数据中，论文必须讨论这一风险并尝试评估其影响。

### 整体判断原则

一篇高质量的"使用 GPT-4 的论文"应该让读者获得超越"GPT-4 在 X 任务上表现很好/不好"的洞见。它应该帮助我们理解为什么、在什么条件下、通过什么机制，以及这些发现如何指导未来的研究方向。

## 8. 给 AI 审稿系统的建议：NLP 论文的特殊审查点

### AI 审稿助手应重点关注的 NLP 特殊检查项

1. **Responsible NLP Checklist 自动审查**：检查 checklist 每一项的填写是否合理。标记所有项均回答"Yes"但缺乏具体说明的情况；检查声称使用了人工标注但未填写 D 部分的不一致；验证声称使用了 AI 辅助工具但 E 部分为空的矛盾。

2. **Limitations Section 质量评估**：检测 Limitations section 是否存在、是否实质性地讨论了局限性。纯粹形式化的 limitations（如"未来我们会做更多实验"）应被标记。对比论文声明的贡献范围与 limitations 的讨论范围是否逻辑自洽。

3. **LLM 使用规范性检查**：自动检测论文中使用的 LLM API 是否标注了版本号和调用时间；检查是否报告了 prompt 模板全文（至少在附录中）；验证 LLM 评估场景是否有人工验证步骤；标记潜在的数据污染风险。

4. **多语言评估覆盖度分析**：自动统计论文涉及的语言种类、语系分布和资源水平分布；标记声称"多语言"但仅覆盖高资源欧洲语言的论文；检查不同语言上的实验结果是否有显著差异及讨论。

5. **Human Evaluation 规范检查**：自动检测是否报告了标注者数量、inter-annotator agreement、标注指南、报酬信息和伦理审查；标记样本量过小（如少于 100 个实例）的人工评估。

6. **统计报告完整性**：检查是否报告了显著性检验和多次运行统计量；标记只报告单次运行最优结果的情况；检查消融实验中各组件贡献的逻辑一致性。

7. **引用时效性与完整性**：结合论文所属子领域，自动检索最近 12 个月内的相关高影响力论文，标记可能遗漏的关键参考文献。特别关注是否遗漏了直接竞争方法。

### AI 审稿系统在 NLP 领域的特殊局限

1. **语言学直觉的缺失**：一篇关于句法分析的论文其方法是否具有语言学上的合理性、一个面向形态丰富语言的方法是否恰当地处理了形态变化——这些需要深厚的语言学素养，当前 AI 难以替代。

2. **跨领域影响力判断**：NLP 论文常常对其他领域（如计算社会学、数字人文、生物医学）产生重大影响，判断这种跨领域影响力需要广泛的学术视野。

3. **伦理敏感性评估**：NLP 研究涉及语言这一人类核心能力，其伦理影响比许多其他 AI 子领域更为直接和深远。判断一项研究是否可能加剧偏见、侵犯隐私或造成文化伤害，需要细腻的人文素养。

4. **审稿偏差的自我校正**：ARR 列出的 17 种常见审稿偏差（H1-H17）中有多项与 NLP 特别相关，如对非英语研究的偏见（H10）、用后见之明否定负面结果（H1）、要求论文达到 SOTA 而论文并未声称如此（H5）。AI 系统需要内建这些偏差检测能力。

**核心建议**：针对 NLP 论文的 AI 审稿助手应在通用论文审查能力之上，内嵌 ARR 的 Responsible NLP Checklist 校验逻辑、LLM 使用规范审查模块和多语言评估标准检查模块。输出时应明确标注每个发现对应的 ARR 审稿指南条目编号（如 M1、R5、H10 等），以便人类审稿人快速定位并做出最终判断。AI 始终应作为辅助工具，将审稿人从机械性检查中解放出来，让他们把精力集中在需要专业判断力的高层次评估上。
