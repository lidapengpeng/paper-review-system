# 会议审稿核心要点精粹

从 10 位顶会审稿人视角文档中提炼的核心审稿要点。Agent 审稿时根据 `--venue` 加载对应章节。

---

## CVPR / ICCV

**审稿优先级**: 新颖性(35%) > 实验(30%) > 清晰度(20%) > 影响力(15%)

**Agent A 侧重点**:
- "30 秒规则"：图表密度高、引用新、结构清晰 = 质量好
- 创新判定：新问题视角 > 新范式 > 新理论理解 > 新能力边界
- 警惕"增量式改进伪装成创新"：仅加模块、简单任务迁移、纯堆规模

**Agent B 侧重点**:
- 公式符号全文一致性检查
- 注意力机制类论文：检查复杂度分析是否合理

**Agent C 侧重点**:
- 必须基线（2024-2025）：检测(DINO-DETR, YOLOv8+)，分割(Mask2Former, SAM)，生成(DiT, SD3)
- 必须数据集：COCO, ImageNet, ADE20K, Cityscapes
- 必须指标：AP/mIoU + Params + FLOPs + FPS
- 检查训练细节中隐藏的不公平比较（数据增强、预训练权重、训练epoch差异）
- 消融实验算术一致性检查

**Agent D 侧重点**:
- 图表质量极其重要：模糊、低分辨率是扣分项
- 可视化结果是否cherry-pick

**ICCV 额外关注**:
- 3D 论文：坐标系陷阱、评估协议不一致（Eigen vs Garg crop）
- 点云密度偏差、Chamfer Distance 方向性

## NeurIPS

**审稿优先级**: 理论贡献(30%) > 实验严谨性(25%) > 新颖性(25%) > 可复现性(20%)

**Agent A 侧重点**:
- "5 年测试"：5 年后人们还会引用这篇论文吗？
- "教科书测试"：这个方法/理论能写进教科书吗？
- Broader Impact Statement 是否实质性（非模板句）

**Agent B 侧重点**:
- 理论论文：证明逐行验证，检查 union bound 中的 log 因子、收敛类型混淆
- 假设是否过强，是否脱离实际

**Agent C 侧重点**:
- **必须有 error bars**（多次运行均值 ± 标准差）
- 合成实验必须与理论预测吻合
- Reproducibility Checklist 合规

**评分映射**: NeurIPS 1-10 → 系统 1-5 (8-10→5, 6-7→4, 5→3, 3-4→2, 1-2→1)

---

## ICML

**审稿优先级**: 数学正确性(35%) > 技术深度(25%) > 假设合理性(20%) > 实验(20%)

**Agent B 侧重点（核心）**:
- 证明审查方法论：自顶向下结构检查 → 逐行验证 → 反例构造 → 量纲分析 → 特殊情形验证
- 常见证明错误：测度论基础错误、累积不等式误差、鞍点与极值点混淆、隐含假设、大O表示法滥用
- "太好了不真实"原则：结果超越已知下界时要格外警惕
- 极端情形检查、常数传播追踪

**Agent C 侧重点**:
- 理论论文：合成实验是否验证了理论预测
- 实验论文：是否有形式化验证（不能纯靠 empirical）

---

## ICLR

**审稿优先级**: 理解深度(30%) > 新颖性(25%) > 实验深度(25%) > 清晰度(20%)

**核心问题**: "这篇论文是否加深了我们对深度学习某个方面的理解？"

**Agent A 侧重点**:
- 偏好方向：理解 DL 现象（Lottery Tickets, Grokking, Scaling Laws, ICL）
- Foundation model 论文：不能仅仅是"更大更好"，需要 insight

**Agent C 侧重点**:
- **深度 > 广度**：3 个 benchmark 做透 > 10 个 benchmark 只报数字
- 推荐实验结构：主实验 + 详细消融 + 分析实验 + 失败案例

**审稿质量**: OpenReview 公开审稿，评审意见本身的质量也被社区评判
- 必须区分 Major / Minor 问题
- 需要明确表达不确定性

---

## ACL / EMNLP

**审稿优先级**: 可靠性(30%) > 创新性(25%) > 可复现性(25%) > 伦理合规(20%)

**Agent C 侧重点**:
- **必须包含 LLM 基线**（zero-shot / few-shot GPT-4 / Claude）
- 多语言评估：typologically diverse languages（不能只有英语+中文）
- Human evaluation：annotator 资质、IAA、报酬、伦理审查
- 统计检验：bootstrap test、Bonferroni 校正、effect size

**Agent D 侧重点**:
- **Limitations section 是强制性的**（缺失 = desk reject）
- Responsible NLP Checklist (A-E) 必须完整
- 使用 LLM 时：需声明 API 版本、成本、数据污染分析

**LLM 论文评判**:
- 可接受：LLM 作为基线、分析工具、数据增强
- 不可接受：仅用 API 调用无深入分析、未披露版本、忽略数据污染

---

## AAAI

**审稿优先级**: 影响力(30%) > 技术可靠性(25%) > 新颖性(25%) > 清晰度(20%)

**特殊注意**:
- 覆盖 AI 全子领域，审稿人可能不完全对口
- Two-Phase Review：初审 + 讨论后复审
- 对应用论文友好（但需要技术深度 + 领域洞察）
- 30 分钟初筛三个问题：(1) 问题是否有意义？(2) 方法是否有道理？(3) 实验是否支撑结论？

---

## KDD

**审稿优先级**: 可扩展性(30%) > 实际影响(25%) > 方法论(25%) > 实验(20%)

**Agent C 侧重点**:
- **可扩展性实验是必须的**（时间/空间随数据量的变化曲线）
- Research Track：标准学术实验
- ADS Track：需要在线 A/B test 结果、服务延迟 <50ms、业务指标
- 推荐系统基线：SASRec, LightGCN, SimGCL
- 图挖掘基线：OGB benchmarks

---

## MICCAI

**审稿优先级**: 临床价值(25%) > 医学实验严谨性(25%) > 技术创新(20%) > 可复现性(15%) > 伦理(15%)

**Agent C 侧重点**:
- **nnU-Net 是必须基线**
- 交叉验证（数据量 <500 时）
- 与 FDA 批准系统对比（如适用）
- 医学数据集：MSD, BraTS, ACDC, Camelyon, Cholec80
- 医学指标：Dice + Hausdorff Distance + Sensitivity + Specificity

**致命问题检测**:
- 数据泄露（训练/测试重叠）
- 未公开的预处理
- 无 IRB 批准
- 过度声称临床价值

---

## ECCV

**审稿优先级**: 方法论创新(30%) > 实验(25%) > 写作(25%) > 影响力(20%)

**特殊偏好**:
- 理论深度 > 工程优化
- "简单但有效"需要有原理解释（三个检验：有原理支撑？多维度有效？有深入分析？）
- 对跨领域研究更开放

---

## ICME / ACM MM / WACV

**ICME**: 6 页限制，效率指标（FPS/Params）较重要，接收率~30%
**ACM MM**: 鼓励多模态（视觉+音频+文本），注重应用价值
**WACV**: 比 CVPR/ICCV 更友好，接受扎实的增量工作，接收率~40%
